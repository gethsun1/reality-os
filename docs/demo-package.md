# Demo Package

## Pitch Deck Outline (10 slides)
1. Title — RealityOS: IP-native reality competitions on Story Protocol.
2. Problem — Fans lack ownership; shows lack transparent attribution.
3. Solution — On-chain IP registration, authenticity, and fan economics.
4. Product — RealityShowIP contracts, Coinbase/Dynamic wallets, dashboards.
5. Market — Reality TV + UGC; unlock global co-creation and royalties.
6. Traction Plan — Launch on Story testnet, community pilots, creator collabs.
7. Token Model — Franchise token (RFT) + royalty fractions per asset.
8. Go-To-Market — Partner shows, fan staking campaigns, Yakoa-verified drops.
9. Roadmap — v1 launch, mobile companion, secondary royalties, governance.
10. Ask — Deploy on Story, seek grants/partners, invite pilot producers.

## 2-Min Demo Script
1. Open RealityOS dashboard; connect Coinbase Embedded wallet.
2. Verify a user-submitted clip via Yakoa → score displayed → click “Register as IP”.
3. Contestant asset minted; mint fractional royalty tokens to producers.
4. Create an episode tied to the contestant; upload metadata to IPFS.
5. Fan submits storyline contribution; Yakoa verifies and registers as IP.
6. Fans stake franchise tokens to boost contestant ranking and choose challenges.
7. Show royalty token vesting view; simulate purchase.
8. Conclude with addresses.json and Dune schema references.

## Dune Dashboard Narrative
- KPI 1: IP creation rate — contestants, episodes, contributions per day.
- KPI 2: Fan economic participation — franchise stakes + royalty token buys.
- KPI 3: Authenticity funnel — uploads verified by Yakoa vs. rejected.
- KPI 4: Royalty flows — mint/transfer volume per asset, top earners.
- KPI 5: Engagement — votes and contributions weighted by reputation.
- Filters: show/season, contestant, timeframe; toggle authenticity-only assets.
